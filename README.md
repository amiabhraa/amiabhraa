### About

I'm a Computer Science student who cares less about writing code that *runs* and more about writing code that **holds up** — under load, under change, and under scrutiny.

My focus sits at the layer beneath the framework: how a request actually moves through a system, why a particular index makes a query fast, what a scheduler is doing when a thread blocks. I spend deliberate time in **algorithms**, **operating systems**, **networking**, and **database internals**, because abstractions are only useful once you understand what they're hiding.

Right now that curiosity is channelled into backend engineering — **Java**, **Spring Boot**, relational data modeling, and the early stages of **system design** — with the long-term goal of building software that's not just functional, but *deliberate*.
